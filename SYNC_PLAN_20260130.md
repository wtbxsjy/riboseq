#### 关键脚本功能详解

**14_ribotish_to_gencode.sh**
```bash
用途：Ribo-TISH predict 输出 → GENCODE 兼容格式
输入文件：
  - ${SAMPLE}_pred.txt (Ribo-TISH predict 输出)
  - genome.fa (基因组序列，用于序列提取)
输出文件：
  - ${SAMPLE}.gencode.fa (序列头: >ORF_NAME--STUDY_ID)
  - ${SAMPLE}.gencode.bed (1-based 坐标)
参数：
  --min-length 16 (最小 ORF 长度，氨基酸)
调用：
  singularity exec biopython:1.81 python3 ribotish_to_gencode.py
```

**15_ribotricer_to_gencode.sh**
```bash
用途：Ribotricer translating_ORFs.tsv → GENCODE 兼容格式
输入文件：
  - ${SAMPLE}_translating_ORFs.tsv (Ribotricer detect-orfs 输出)
  - genome.fa (基因组序列)
输出文件：
  - ${SAMPLE}.gencode.fa
  - ${SAMPLE}.gencode.bed
参数：
  --min-length 16
  --min-phase-score 0.5 (Ribotricer phase score 过滤)
调用：
  singularity exec biopython:1.81 python3 ribotricer_to_gencode.py
```

**16_gencode_orf_mapper.sh**
```bash
用途：多工具/多样本 ORF 统一注释、去冗余、Ensembl 映射
输入文件：
  - merged_all_tools.fa (所有工具的合并 FASTA)
  - merged_all_tools.bed (所有工具的合并 BED)
  - Ensembl 注释目录 (PROTEOME_FASTA, TRANSCRIPTOME_FASTA, SORTED_TRANSCRIPTOME_GTF 等)
输出文件：
  - ${PROJECT}.orfs.fa (最终 ORF 序列)
  - ${PROJECT}.orfs.bed (最终坐标)
  - ${PROJECT}.orfs.gtf ⭐ (用于定量的 GTF，包括工具来源信息)
  - ${PROJECT}.orfs.out ⭐ (ORF 特征表，包含工具检测信息矩阵)
  - ${PROJECT}.altmapped (备选映射)
  - ${PROJECT}.unmapped (未映射的 ORF)
参数：
  --min-length 16
  --collapse-threshold 0.78 (宽松，多工具比较)
  --collapse-method longest_string (序列相似度去冗余)
调用：
  singularity exec gencode-orf-mapper:custom python3 ORF_mapper_to_GENCODE_v1.1.py
```

// 在所有 sORF 预测完成后

if (!params.skip_gencode_annotation && params.ensembl_dir) {
    GENCODE_ANNOTATION(
        RIBOTISH_PREDICT_INDIVIDUAL.out.prediction,      // Ribo-TISH
        RIBOTRICER_DETECTORFS.out.orfs,                  // Ribotricer
        ch_fasta,
        params.ensembl_dir,
        params.orf_collapse_threshold
    )
    ch_versions = ch_versions.mix(GENCODE_ANNOTATION.out.versions)
    
    // 存储 GTF 和 OUT 用于后续分析
    ch_orfs_gtf = GENCODE_ANNOTATION.out.orfs_gtf
    ch_orfs_out = GENCODE_ANNOTATION.out.orfs_out
} else {
    log.warn "GENCODE annotation skipped (set ensembl_dir to enable)"
    ch_orfs_gtf = Channel.empty()
    ch_orfs_out = Channel.empty()
}

if (!params.skip_orf_quantification && ch_orfs_gtf) {
    QUANTIFY_ORFS(
        ch_orfs_gtf,
        RIBOSEQC_POSTFILTER.out.bedgraph_dir
    )
    ch_versions = ch_versions.mix(QUANTIFY_ORFS.out.versions)
    ch_multiqc_files = ch_multiqc_files.mix(QUANTIFY_ORFS.out.multiqc_files)
}

if (!params.skip_tool_comparison && ch_orfs_out) {
    TOOL_COMPARISON(
        ch_orfs_out,
        "ORFquant,RiboTISH,Ribotricer"  // 实际工具列表
    )
    ch_versions = ch_versions.mix(TOOL_COMPARISON.out.versions)
    ch_multiqc_files = ch_multiqc_files.mix(TOOL_COMPARISON.out.multiqc_files)
}

emit:
    ...
    orfs_gtf = ch_orfs_gtf              // 新增输出
    orfs_out = ch_orfs_out              // 新增输出
    quantification_matrix = QUANTIFY_ORFS.out.counts_matrix  // (可选)
```

## 6. 关键考虑事项

### 数据流管理
- **BAM 输入**：确保 GENCODE 注释步骤接收经过 sORF_BAM_FILTER 的 BAM
- **多工具合并**：按样本分组后再合并所有工具的输出
- **GTF 输出**：GTF 应包含工具来源信息（用于后续过滤）

### 性能优化
- **并行化格式转换**：多个工具的 FASTA/BED 转换可并行进行
- **Ensembl 缓存**：Ensembl 数据应该只下载一次，在流程开始前准备
- **ORF Mapper 内存**：大型数据集（>10K ORF）可能需要 16+ GB 内存

### 容器镜像
- **biopython:1.81**：轻量级，已被广泛使用（～200 MB）
- **gencode-orf-mapper**：自定义镜像，可能需要从源代码构建或 BioConda 获取

### 参数暴露
- 所有脚本参数应该暴露为 Nextflow 参数（`params.*`）
- 提供合理的默认值
- 在 `nextflow_schema.json` 中添加帮助文本和约束

### 关键输入文件格式

**Ribo-TISH predict 输出** (`*_pred.txt`)
```
transcript_id   strand  orf_id  start_codon_pos  orf_start  orf_end  orf_length
NM_001234   +   ORF.1   100   100   450   350
```

**Ribotricer translating_ORFs TSV** (`*_translating_ORFs.tsv`)
```
transcript_id   strand  orf_id  orf_start  orf_end  orf_length  fscore  pscore

```

**GENCODE 格式 FASTA** (`*.gencode.fa`)
```
>ORF_NAME--STUDY_ID
MPPQVALITQLCLGDSSCSPS...
```

**GENCODE 格式 BED** (`*.gencode.bed`)
```
